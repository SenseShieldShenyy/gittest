本demo分模块实现不同的身份认证方式
1 login 实现的是随机数挑战响应的方式实现身份认证，浏览器端将特定的随机数，通过调用深思结构，使用深思加密锁证书链进行签名之后。将数据返回服务端进行验签验签通过则身份认证成功
  使用此demo若要验证通过需要
  1.1修改webform1.aspx.cs 处已经固定好的证书链。
  1.2修改login.aspx.cs处已经固定的许可ID 为锁内已经存在的任意许可id
 
2 shell_num 实现的是通过验证加密锁号的方式，实现身份认证。需要
  2.1 修改chek_shell_num.aspx.cs 处已经固定好的加密锁号

3 get_pubdata 实现的是通过读取加密锁内公开区的数据来代替账号密码实现身份认证，调通这个部分需要：
  3.1 修改get_pubdata.js 中已经固定号的许可ID ，确保对应许可中有公开区数据是存储着账号和密码
  3.2 demo中公开区的数据 前19个字符为账号 shenyy@sense.com.cn 后10个字符为密码 1234567890
  3.3 如果需要验证通过，使用则需要根据自己的账号密码修改代码中 取得账号密码的规则，并将验证的账号密码修改为自己的账号密码
